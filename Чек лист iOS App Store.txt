1.Проверка безопасности.

1.1 Приложение не содержит нежелательного контента (оскорбительного, бесчувственного, расстраивающего, вызывающего отвращение или исключительно плохого вкуса).
1.2 Приложение имеет механизм фильтрации нежелательных материалов.
1.3 Возможность сообщения об оскорбительном контенте и блокировки пользователей.
1.4 Контактная информация разработчика релевантна и актуальна. 
1.5 При определения приложения в детскую категорию, оно не содержит ссылки из других приложений, возможность покупки и другие отвлекающие факторы для детей.
1.6 Исключен риск причинить физический вред пользователю.
1.7 Контактная информация разработчика релевантна и актуальна.
1.8 Пользовательская информация недоступна для несанкционированного использования, раскрытия или доступа третьим лицам.
1.9 Приложения для сообщения о предполагаемой преступной деятельности должны привлекать местные правоохранительные органы и могут предлагаться только в странах, где такое участие активно.

2.Проверка производительности.

2.1 Приложение готово и не содержит ошибок.
2.2 Приложение не является бета-версией, демонстрацией или пробной версией.
2.3 Метаданные точны. Никаких скрытых или недокументированных функций.
2.4 Совместим с iPad и iPhone под управлением последней версии iOS
2.5 Использует только общедоступные API.
2.6 Совместимость с IPv6
2.7 Запрашивает явное согласие пользователя и обеспечивает четкую индикацию при записи активности пользователя (камера, микрофон, запись экрана и др.)

3.Бизнес-требования.
3.1 Приложение не упоминает другие поддерживаемые платформы
3.2 Покупки в приложении можно делать только через систему Apple IAP.
3.3 Подписки должны обеспечивать постоянную ценность для покупателя (т. е. давать новый контент)
3.4 Пользователи подписки должны иметь возможность легко обновлять или возвращать старую версию.
3.5 Не следует использовать коды содержимого для разблокировки нового содержимого.
3.6 Не должен быть каталог других приложений (не конкурировать с App Store).
3.7 Страховые приложения должны быть бесплатными
3.8 Не должно ограничиваться местоположением или перевозчиком.

4.Дизайн
4.1 Дизайн оригинальный. Не является подражанием другим популярным приложениям.
4.2 В приложении должна быть хоть какая-то функциональность. Оно не может состоять из одной странички и текста, не может быть просто песней, фильмом или книгой.
4.3 Соответствует рекомендациям Apple по интерфейсу пользователя.
4.4 Это не переработанный веб-сайт.
 
5. Юридические вопросы.

5.1 Имеет Политику конфиденциальности (если хранит пользовательские данные).
5.2 Перед передачей личных данных получает разрешение от пользователя.
5.3 Приложение запрашивает доступ только к данным, относящимся к основным функциям приложения.
5.4 Данные из HomeKit API не используются для рекламы
5.5 Не использует и не раскрывает третьим лицам данные, собранные в контексте исследований здоровья, фитнеса и медицинских исследований.
5.6 Если в приложении есть азартные игры, должна быть лицензия и разрешение в местах, где используется приложение.
