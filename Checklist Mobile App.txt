1. Характеристики устройства.

1.1 Можно ли установить приложение на устройство.

Приложение ведет себя как задумано:
1.2 при входящем звонке
1.3 при входящей SMS
1.4 при подключении зарядного устройства
1.5 при отключении зарядного устройства
1.6 при переходе в спящий режим
1.7 при выходе из спящего режима
1.8 при разблокировке экрана
1.9 если устройство наклонено
1.10 при встряхивании устройства
1.11 при push-уведомлении из встроенных приложений (например, календарь)
1.12 при push-уведомлении из других приложений (Instagram, Twitter, VK)
1.13 при сообщении о недостаточном заряде батареи
1.14 при выключенном звуке
1.15 при авиарежиме.

1.16 Правильно ли приложение взаимодействует с датчиком GPS (включение / выключение, получение данные GPS).
1.17 Определены ли функции всех кнопок или клавиш на устройстве для этого приложения.
1.18 Убедиться, что кнопки или клавиши, не ассоциированные с функциями приложения, не вызывают неожиданного поведения при активации.
1.19 Если на устройстве доступна физическая кнопка "назад", переводит ли она пользователя на предыдущий экран.
1.20 Если на устройстве доступна физическая кнопка "меню", открывает ли она меню приложения.
1.21 Если на устройстве доступна физическая кнопка "домой", переносит ли она пользователя на главный экран устройства, если приложение запущено.
1.22 Если на устройстве доступна физическая кнопка "поиск", открывает ли она поиск внутри приложения.
1.23 Можно ли удалить приложение с устройства.
1.24 Работает ли приложение должным образом после установки.
1.25 Можно ли найти приложение в магазине приложений (после одобрения в магазине).
1.26 Может ли приложение переключаться на другие приложения устройства через режим мультизадачности (если должно).
1.27 Все ли активные области на экране/кнопки работают в приложении, если используется защитный экран.

2. Сетевые характеристики.

2.1 Соответствует ли поведение приложения желаемому, если оно подключено к Интернету через Wi-Fi.
2.2 Соответствует ли поведение приложения желаемому, если оно подключено к Интернету через 2G/3G/4G.
2.3 Соответствует ли поведение приложения желаемому, если сеть недоступна.
2.4 Возобновляет ли приложение работу, когда снова получает доступ к сети после прерванного доступа.
2.5 Продолжает ли приложение корректно работать, если оно привязано или каким-либо другим образом соединено с другим устройством?
2.6 Что происходит, если приложение переключается между сетями (Wi-Fi, 3G, 2G)?
2.7 Использует ли приложение стандартные сетевые порты (Почта: 25, 143, 465, 993 или 995, HTTP: 80 или 443, SFTP: 22) для удаленных подключений: некоторые провайдеры блокируют отдельные порты.

3. Характеристики приложения.

3.1 Проверялось ли приложение на различных устройствах и на разных версиях ОС.
3.2 Проверка стабильности: если в приложении есть список (например, изображений), попробовать пролистать его на большой скорости.
3.3 Проверка стабильности: если в приложении есть списки (например, изображений), попробовать пролистать их до позиции перед первым изображением и после последнего.
3.4 Запрещается ли скачивание приложения, если оно превышает допустимый в ОС размер для загрузки через мобильный интернет.
3.5 Интеграция: правильно ли приложение подключается к разным социальным сетям (LinkedIn, twitter, facebook и т. Д.) Или к любому другому приложению или сайту.
3.6 Приложение не мешает другим приложениям в фоновом режиме или режиме многозадачности(использование GPS, воспроизведение музыки и т. д.).
3.7 Пользователь может распечатать из приложения (если применимо).
3.8 Опция поиска в приложении отображает релевантные результаты.
3.9 Проверка наоболее распространенных жестов, используемых для управления приложением.
3.10 Что произойдет при выборе нескольких опций одновременно (незапланированный мультитач – например, выбор нескольких контактов из записной книжки разом).
3.11 Название приложения должно быть понятным.
3.12 Есть ли в приложении ограничение и очистка кэша.
3.13 Перезагрузка данных от удаленного сервиса сконфигурирована так, чтобы не вызывать проблем с производительностью на стороне сервера (ручная перезагрузка может снизить количество обращений к серверу)
3.14 Переходит ли приложение в спящий режим при работе в фоновом режиме (предотвращение разрядки батареи).
3.15 Запрашивает ли приложение разрешение на доступ к определенной функции телефона. (Например, календарь, контакты, местоположение, GPS и т. д.)

4. Проверка пользовательского интерфейса

4.1 Элементы управления максимально ненавязчивы, наппример за счет их исчезновения, если они не используются какое-то время.
4.2 Пользователь может вернуться на предыдущий экран (например, нажав "Назад" или "Отмена").
4.3.Главная функция приложения очевидна и говорит сама для себя.
4.4 Функция, которую, скорее всего, будет использовать пользователь, подсвечена (к примеру, в iOS голубая кнопка обозначает действие по умолчанию/наиболее вероятное действие).
4.5 Действия пользователя минимизированы, используются выпадающие списки или табличное представление, где пользователь может сделать выбор, вместо ручного ввода данных.
4.6 У пользователя нет возможности хранить данные локально, вне "песочницы" приложения.
4.7 Пользователь не имеет возможности просматривать уровни доступа для файлов.
4.8 Если есть длинный список для прокрутки, предоставлена возможность поиска по списку.
4.9 При низкой производительности есть значок состояния выполнения ("Загрузка..."), желательно с понятным пользователю сообщением.
4.10 В случае "живой" фильтрации данных при вводе поискового запроса проверить производительность приложения.
4.11 Внешний вид кнопок, выполняющих стандартные действия, привычен пользователю (например: обновить, упорядочить, удалить, ответить, вернуться и т. д.).
4.12 Стандартные иконки не используются для функций, для которых они в норме не применяются.
4.13 Приложение реагирует на все изменения ориентации устройства в соответствии с дизайном.
4.14 Активные элементы экрана должны быть примерно 7х7 мм по размеру. Используйте плотность пикселей целевого устройства, чтобы вычислить количество нужных пикселей.
4.15 Жесты в приложении, которые имеют стандартное значение, не переопределены (пример: смахивание сверху вниз включает центр уведомлений).
4.16 Требование входа в систему откладывается в приложении на максимально долгий срок.
4.17 Если приложение было остановлено в неожиданное время, пользовательские данные сохранены локально и доступны при запуске.
4.18 При удалении документов пользователь предупреждается о последствиях.
4.19 Клавиатура подстраивается под ожидаемый ввод (к примеру, цифры/буквы).
4.20 Неактивные кнопки четко отличаются от активных кнопок.
